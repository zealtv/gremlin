# s54-memory-field-use field report

Status: waiting for real use.

The protocol-level memory loop has passed acceptance, but this stitch should
not be tied until the system has been used across ordinary sessions with a real
model making real distillation choices. Synthetic tests prove wiring; they do
not prove judgment, restraint, or day-to-day user experience.

## How To Run The Field Test

Use a personal gremlin for normal work. After each session, fill one row in the
session log below.

Recommended minimum:

1. three `/new` closures from ordinary work;
2. one `/discard` closure for temporary or sensitive work;
3. one correction to an existing finding;
4. one session where nothing durable should be remembered.

Do not tune the implementation during the field test. Record friction and turn
important observations into follow-up Loom stitches after the pass.

## Scoring

Use `good`, `mixed`, or `poor` for each dimension.

- **Salience:** remembered things that actually help later.
- **Restraint:** avoided incidental, temporary, or sensitive details.
- **Traceability:** finding explains why it exists and where it came from.
- **Distillation:** finding is shorter and more useful than the transcript.
- **Retrieval:** relevant memory can be found without always loading everything.
- **Promotion:** clear which findings deserve `context/`.
- **Correction:** contradictions revise old findings instead of duplicating.
- **User experience:** commands and visible review outcome feel clear in daily use.
