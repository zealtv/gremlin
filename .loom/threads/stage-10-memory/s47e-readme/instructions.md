# s47e-readme

*(scribble repo)*

README documents how scribble composes — disclosure layers and the dream-as-item recipe. Durable explanation that lets gremlin (or any consumer) wire it up correctly without scribble depending on any of them.

## Outcome

`README.md` gains two new sections (or expands existing ones):

1. **Disclosure layers** — how a consumer chooses what to load:
   - `findings/INDEX.md` — always-loadable, cheap, regenerated by `scribble.sh index`.
   - `findings/<id>/finding.md` — loaded on lookup (e.g. via `scribble.sh fetch`).
   - `findings/<id>/context.md` — deep dive only.

2. **Composition** — how scribble plays with sibling protocols (nestlings, groundhog, loom) without depending on any of them:
   - Capture: any process can write candidates via `scribble.sh capture <id>` from a pipe.
   - Scheduling: a scheduler such as groundhog can fire a `dream` instruction into a tender's inbox; the tender reads `.scribble/dream.md` and runs the dream pass against scribble's surfaces. Scribble itself does not schedule and does not tend.
   - Lookup: any agent can call `scribble.sh fetch <words>` to discover relevant findings before composing a prompt.
   - Scribble does not call a model. The dream brief is opinion; the CLI is mechanics. Tool- and model-agnostic.

## Scope

- README only. No code change.
- Land after `s47a`–`s47d` so the doc references real commands.
