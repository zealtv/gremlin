# readme-docs-restructure

Goal: simplify the public docs without losing the detailed protocol material.

The README should become a short entry point with:

- getting started
- basic customization
- the core mental model
- links to deeper docs

Move detailed mechanics into docs after thinking through the split. Candidate
docs:

- protocol reference: layout, loops, transcript, skills, tools, models, data flow
- composition: multiple gremlins, delegation, shared context, sandboxing

Do not implement this directly from the goal stitch. Before editing README or
creating docs, decompose this into child stitches with clear acceptance criteria.
